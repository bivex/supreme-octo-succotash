-R, --report=<string>         Generate a report from data collected during a
                              previous analysis.


Generates a formatted data report with the specified type and action options.

 Usage: advisor --report=<string> [--action-option] [--global-option] [[--]
        <target> [<target options>]]

        <string> is one of the following reports:

            survey        - Report results of a Survey analysis.
            all           - Generate an interactive HTML report for all collected results.
            annotations   - Report the annotations in the source code.
            custom        - Generate a custom report
            dependencies  - Report results of a Dependencies analysis.
            hotspots      - Report results of a Survey analysis.
            joined        - Combine results for several analyses into a single report.
            map           - Report results of a Memory Access Patterns analysis.
            roofline      - Report results of a Roofline analysis.
            roofs         - Report roof values.
            suitability   - Report results of a Suitability analysis.
            summary       - Report the analysis summary.
            threads       - Report on threads.
            top-down      - Report results of a Survey analysis in top-down view.
            tripcounts    - Add trip counts data to a Survey report.

                              with --format=csv.

## Основные отчеты

### 1. Survey (общий анализ производительности)
```bash
advixe-cl --report=survey --project-dir=./my_results --format=text --report-output=survey.txt
```

### 2. Dependencies (дедлоки, race conditions)
```bash
advixe-cl --report=dependencies --project-dir=./my_results --format=text --report-output=dependencies.txt
```

### 3. Map (Memory Access Patterns)
```bash
advixe-cl --report=map --project-dir=./my_results --format=text --report-output=map.txt
```

### 4. Suitability (анализ потоков)
```bash
advixe-cl --report=suitability --project-dir=./my_results --format=text --report-output=suitability.txt
```

### 5. Threads (информация о потоках)
```bash
advixe-cl --report=threads --project-dir=./my_results --format=text --report-output=threads.txt
```

### 6. Summary (краткая сводка)
```bash
advixe-cl --report=summary --project-dir=./my_results --format=text --report-output=summary.txt
```

## Расширенные опции

### С callstack (стеком вызовов)
```bash
advixe-cl --report=survey --project-dir=./my_results --display-callstack --format=text --report-output=survey_with_stack.txt
```

### Только функции (без петель)
```bash
advixe-cl --report=survey --project-dir=./my_results --show-functions --no-show-loops --format=text --report-output=functions.txt
```

### Только петли
```bash
advixe-cl --report=survey --project-dir=./my_results --show-loops --no-show-functions --format=text --report-output=loops.txt
```

### Со всеми колонками
```bash
advixe-cl --report=survey --project-dir=./my_results --show-all-columns --format=text --report-output=survey_full.txt
```

### Сортировка по времени (по убыванию)
```bash
advixe-cl --report=survey --project-dir=./my_results --sort-desc="Total Time" --format=text --report-output=survey_sorted.txt
```

## CSV формат (для Excel/анализа)

```bash
# Survey в CSV
advixe-cl --report=survey --project-dir=./my_results --format=csv --csv-delimiter=comma --report-output=survey.csv

# Dependencies в CSV
advixe-cl --report=dependencies --project-dir=./my_results --format=csv --report-output=dependencies.csv
```

## Комбинированный отчет

```bash
# Все результаты в одном HTML (интерактивный)
advixe-cl --report=all --project-dir=./my_results --report-output=full_report.html
```

## Для GPU данных

```bash
advixe-cl --report=survey --gpu --project-dir=./my_results --format=text --report-output=gpu_survey.txt
```

**Совет:** Для быстрого просмотра всех результатов начните с:
```bash
advixe-cl --report=summary --project-dir=./my_results --format=text
```
